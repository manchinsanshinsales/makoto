# Antigravity 運用ルール (Agent.md)

このプロジェクト（`makoto`）における Antigravity の行動指針を定義します。

## GitHub 自動プッシュ連携
- **ルール**: ファイルの編集やタスクの完了後、必ず `./autopush.sh` を実行して GitHub へ最新の状態をプッシュすること。
- **目的**: 常に Git リポジトリとリモート（GitHub）の整合性を保ち、ユーザーの手作業をゼロにする。
- **実行手順**:
  1. 編集が完了したら `autopush.sh` を呼び出す。
  2. 引数に変更内容の概要を渡して、適切なコミットメッセージを作成する。
  3. 万が一プッシュが失敗した場合は、ユーザーに報告し、代行を依頼するか調査する。

## 除外ファイルの管理
- `.gitignore` に含まれていない不要なファイルを発見した場合は、即座に追記してリポジトリの軽量化を図る。
